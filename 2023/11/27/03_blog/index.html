如何从0搭建并部署一个自己的博客系统

智锐科创2023-11-272023-11-27

前言

作为一个计算机系的同学，面对行业内日新月异和层出不穷的新技术，养成写博客的习惯非常能够帮助自己加深对知识点的理解。根据费曼学习法，输出是一种效率极高的学习方式，写博客就是这样一种输出的方式，优质的博客能够帮助读者更好的理解某个技术或者解决某个问题。

与此同时，拥有一个个人的博客网站，记录自己的学习笔记和记录生活日常也是一件非常有意义的事情，你可以在博客中记录你的C语言学习笔记，记录你的生活日常，记录你的读书笔记……本次讲座中，我们会带大家一步一步搭建一个和本站一样的博客网站。

1. 什么是静态博客？

本次我们要介绍的博客框架属于静态博客，可以简单理解为没有后台管理系统的博客，不需要连接数据库的博客。

Hexo是一个静态博客框架，它可以将你写的markdown语法渲染成HTML，配合一些主题插件，你只需要关注你的博客内容即可，样式方面可以通过配置文件配置的方式来完成。他的生态非常丰富，有很多大佬写了很多很不错的主题和插件。

同时，他可以通过配置文件轻松实现你DIY的目的，而不需要通过写代码的方式，所以也是十分友好的。

与此相反的是带有后台管理系统的博客，典型的如WordPress，大家寒假有空的话可以研究一下。他是用php语言写得，你写的博客都会存到MySQL数据库里面，同时他也提供了一个后台管理系统，你可以通过他提供的各类UI来操作你的博客。

2. 初探前后端开发

简单理解：

  • 前端：UI + 事件处理 + 发送请求。
  • 后端：提供Api接口，处理数据，返回数据。

任何一个页面都可以简单划分为两个部分，一部分是UI，另一部分则是数据。前者主要由前端负责，后者主要由后端负责。

拿川大教务处举例，你打开教务处的网页，这些网页上的图片、布局、颜色、交互动画等等都由前端完成。但是公告数据、你的账号密码等内容都是存放到数据库的，是有一个专门的后端提供Api接口，供前端进行获取。前端获取后再对数据进行展示。

这里演示下F12获取前端向后端发的请求。

对前后端开发感兴趣的同学可以在寒假去了解一下。

  • 前端开发：
    • 基础：HTML / CSS / JavaScript
    • 进阶：Vue / React
  • 后端开发：
    • Java、Python、Go、php …

Node.js 环境安装

Node.js 是一个JavaScript运行环境，用于运行js代码。本次教程不会太多深入Node相关的知识，所以不用慌，你只需要学会安装即可。

npm是一个包管理工具，可以理解为一个应用商店，各类js的包都可以通过npm进行下载。默认的仓库地址在国外，下载比较慢，所以下面我们需要将npm源修改为淘宝源。如果学过Unix操作系统的同学应该比较好理解。

  1. 打开NodeJs官网：https://nodejs.org/en/download/releases/，可以选择14版本进行安装。
  2. 安装的时候可以自定义路径，尽量不要选择C盘。小Tips：可以在D盘下建一个Dev文件夹，专门安装各类开发环境。
  3. 安装完成后，输入win +R，输入CMD，执行node -v命令，看到版本信息说明安装成功。
  4. 修改npm源为淘宝镜像：npm config set registry https://registry.npm.taobao.org

补充： Yml文件介绍

Yml没什么神奇的，就是一种配置文件，程序通过读取配置文件上的配置，来执行一些相应的操作。

这就类似于如果我们想要用C语言开发一个游戏，假设这个游戏有普通模式、无敌模式、多人模式，我们就可以将用户的选择保存到文件中，这样下次再次打开游戏的时候，就直接读取文件内容后选择对应模式即可。

yml也是这个道理，其它的配置文件还可以是Json、xml、properties等等，大家在以后的学习之路上会慢慢遇到的。

在这里，我们只需要知道，我们通过yml文件进行配置，就可以对我们的网站进行DIY。至于程序内部是怎么读的，怎么解析的，这些都是大佬们写好的，你这个学习阶段不用管，用就好啦~
